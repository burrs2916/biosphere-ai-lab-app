✅ 正确的下一步（必须按顺序）
🔹 Step 1：锁死 Manifest 的“可遍历性”

你现在的 ManifestNode 是正确的，但你必须补齐这三个能力：

impl ManifestNode {
    pub fn children(&self) -> &[ManifestNode];
    pub fn value(&self) -> &Value;
    pub fn kind(&self) -> NodeKind;
}


以及一个 Path 访问模型：

pub type ManifestPath = Vec<usize>;

pub fn get_node(root: &ManifestNode, path: &ManifestPath) -> Option<&ManifestNode>;


这一步不是为了 UI
是为了 Observer 能“走进 Manifest”

🔹 Step 2：实现 Manifest → Linear Perception（极其重要）

UI 不能直接消费树。

第一个 Renderer 不是“画”，而是“展开感知顺序”。

例如：

/ (Group)
  /Entities (Group)
    /0 Scalar "Entity #1"
    /1 Scalar "Entity #2"


变成：

pub struct PerceivedLine {
    pub path: ManifestPath,
    pub depth: usize,
    pub kind: NodeKind,
    pub value: Value,
}

pub struct LinearPerception {
    pub time: u64,
    pub lines: Vec<PerceivedLine>,
}


这是 UI 出生前的最后一层 foundation → ui 过渡层

🔹 Step 3：第一个 UI = LinearPerception Viewer

现在，终于可以写 UI 了。

它可以是：

CLI

TUI

Window

ASCII

Debug Print

但必须遵守：

UI:
  输入：LinearPerception
  输出：人类可读
  不存状态
  不推进时间


这是第一个 UI。
